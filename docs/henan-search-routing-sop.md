# 河南六大领域搜索路由 SOP

适用场景：河南省政府各厅办、各地市政府、驻豫国央企相关监控；六大领域项目机会检索；公众号选题与情报补链。

## 默认搜索顺序

1. `tavily-search`
2. `cn-web-search`
3. `baidu-search`
4. `multi-search-engine`
5. `jina-reader`（正文确认）
6. `ddg-web-search`（兜底）

## 执行原则

- `tavily-search`：作为主搜索入口，优先拿结构化结果与高相关候选链接。
- `cn-web-search`：用于中文网页、地市政府、公众号、行业稿补盲。
- `baidu-search`：用于中文发现层，补热词、本地资讯、转载链路，但不能直接当最终情报源。
- `multi-search-engine`：用于扩搜索口径，切换不同引擎做交叉验证或找漏网信息。
- `jina-reader`：搜索到链接后，用于抓取正文、清洗页面、验证核心事实。
- `ddg-web-search`：仅在其他搜索不可用或效果明显不佳时兜底使用。

## 适合河南六大领域简报的工作流

1. 先用 `tavily-search` 找 3-5 条高相关候选。
2. 再用 `cn-web-search` 补中文本地源和公众号文章。
3. 如需扩展中文发现层，再跑 `baidu-search`。
4. 若权威源仍不够，再用 `multi-search-engine` 切 Bing/360/Sogou/微信搜索补入口。
5. 对命中的关键链接，用 `jina-reader` 或等效正文抓取方式做确认。
6. 若仍找不到，再用 `ddg-web-search` 做最终兜底。

## 输出要求

- 必须区分“发现层”和“确认层”。
- C 类转载/聚合站只能补线索，不能盖过官方源与权威媒体。
- 引用时优先保留：官方源 -> 权威媒体 -> 企业官方 -> 转载聚合。
- 对 403 / 动态页 / 公众号页，优先走 `jina-reader` 或浏览器代抓。
