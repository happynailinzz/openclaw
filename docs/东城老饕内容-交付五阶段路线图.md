# 东城老饕内容-交付五阶段路线图

## 总目标
把当前能力从“会写、会搜、会监控、会输出”升级为“按业务阶段持续产出内容、识别项目、承接交付、放大品牌、沉淀系统”的一套稳定工作流。

---

## 阶段一：认知获客
**目标：** 稳定产出能吸引对的客户与合作线索的内容。

### 核心任务
- 固化个人写作风格与判断口径
- 建立持续选题机制
- 监控热点中的异常值，而不是泛热度
- 围绕主航道稳定输出

### 主航道
- 政企数字化转型
- AI 应用落地
- 算力 / 数据中心 / 信创
- 源网荷储 / 绿色储能 / 风电 / 光伏
- 智慧教育 / 产教融合
- AI 漫剧 / AI 生剧
- 河南项目机会

### 成功标准
- 输出连续不断
- 风格稳定可识别
- 内容开始带来咨询 / 合作线索

---

## 阶段二：内容转化
**目标：** 把“能看”升级成“能点、能读、能转、能留印象”。

### 核心任务
- 公众号 refined HTML 标准化
- 标题、摘要、封面、配图一体化
- 数据图和证据表达替代空洞配图
- 每篇文章至少具备“可直接发”的完成度

### 关键原则
- 排版服务阅读完成率
- 封面服务打开率
- 图表服务论点可信度
- 不做装饰性废图

### 成功标准
- 点击率与完成率提升
- 内容专业感更强
- 文章更像“可成交的专业内容”

---

## 阶段三：线索变项目
**目标：** 把内容流量变成项目机会和可承接交付。

### 核心任务
- 建立“政策/动作 → 场景 → 项目形态 → 牵头单位 → 资金口径”映射
- 将高价值内容同步抽取为项目机会
- 快速生成咨询框架 / 服务框架 / 交付草案
- 建立项目级跟踪和判断机制

### 典型输出
- 项目机会简报
- 客户可做项目清单
- 场景落地方案草案
- 咨询服务切入口

### 成功标准
- 内容不只停留在传播层
- 能导出真实合作话题
- 有复用交付模板

---

## 阶段四：品牌势能
**目标：** 从“有内容”升级到“被行业识别”。

### 核心任务
- 长文拆成多平台内容矩阵
- 建立 GEO / AI 搜索可引用结构
- 形成固定栏目与专题线
- 补齐朋友圈、群发、短视频口播等触点

### 重点方向
- 河南项目机会简报
- AI 生剧 / AI 内容工业化观察
- 源网荷储 / 绿色储能项目窗口观察
- 政企数字化落地点评

### 成功标准
- 内容开始形成栏目感
- 私域与外部搜索入口都能看到你
- 行业内识别度提升

---

## 阶段五：系统化与护城河
**目标：** 把自己从“高手工输出”变成“可复制系统”。

### 核心任务
- 把稳定有效的方法封装为 SOP / skill
- 建立质量评估机制
- 打通 Obsidian / MEMORY / Notion / Get 笔记 / 本地文件的知识流转
- 建立复盘与升级机制

### 判断标准
- 哪个 SOP 真提升结果
- 哪个 skill 值得维护
- 哪些流程应继续封装
- 哪些流程应淘汰

### 成功标准
- 输出不依赖临场状态
- 方法持续沉淀
- 业务越来越像系统，不像临时拼装

---

## 推荐升级顺序
### 第一梯队
1. 写作风格引擎
2. 选题规划引擎
3. 热点异常值识别引擎
4. 项目机会映射引擎

### 第二梯队
5. 公众号 refined HTML 引擎
6. 图表 / 配图证据化引擎
7. 长文拆矩阵引擎

### 第三梯队
8. 项目管理与风险跟踪引擎
9. 知识流转与归档引擎
10. Skill / SOP 评估与创造引擎

---

## 一句话原则
先把“内容获客 + 项目判断”做硬，
再把“内容包装 + 矩阵分发”做强，
最后把“交付管理 + 系统沉淀”做厚。
