AI时代，最先被重做的，可能真的是公司里的中层

作者：余炜勋
来源：外部文章观点整合 + 延伸分析

这两天看到两篇文章，放在一起看，挺有意思。

一篇是陈航讲的：“未来的公司没有文档、没有中层。”
一篇是刘润讲的：“公司中层可能即将消失。”

如果只看标题，很容易把它们理解成那种耸人听闻的管理学判断：

  • • AI来了，中层要没了
  • • 文档没用了，组织要扁平了
  • • 以后老板直接指挥AI和一线员工，公司像科幻片一样运转

但我觉得，这么理解就浅了。

这两篇文章真正戳中的，不是“中层会不会物理消失”，而是一个更现实的问题：

在AI时代，公司里很多原本依赖中层完成的信息整理、任务转译、流程推动、汇报包装，正在被系统、模型和自动化流程接管。

所以真正会被重做的，不是“中层”这两个字，
而是中层过去赖以存在的那部分工作结构。

这才是重点。

一、很多公司的中层，本质上干的不是“管理”，而是“信息搬运”

为什么很多人一听“中层可能消失”，会那么敏感？

因为在大多数公司里，中层本来就不是一个单纯的头衔问题，而是一种组织结构里的中间层。

这层人常见在干什么？

  • • 向上写汇报、做总结、讲结果
  • • 向下拆任务、分动作、盯进度
  • • 左右协调资源、对齐口径、推动协同
  • • 把老板模糊的想法翻译成执行语言
  • • 把一线混乱的信息整理成管理层能看的材料

你会发现，这里面有一大块工作，并不是“管理决策”本身，
而是信息加工和组织传递。

过去为什么这层人重要？

因为没有这层，公司就容易出几个问题：

  • • 上面不知道下面在干嘛
  • • 下面听不懂上面想干嘛
  • • 横向协作靠吼、靠催、靠会
  • • 信息靠人脑记忆和 Word/PPT 流转
  • • 一个简单决策，最后变成多轮汇报、多层转述、多次失真

所以，中层之所以长期存在，不只是因为组织喜欢设层级，
更因为过去的信息系统、协同工具和管理基础设施太弱，很多事情只能靠人肉补上。

说得再直白一点：

很多中层，其实是被低效率组织“养出来”的。

二、AI真正冲击的，不是“人”，而是“中间层的信息垄断”

为什么这次AI会让中层问题突然变得尖锐？

因为它第一次开始系统性地替代中间层最常做的那类工作：

1. 整理信息

过去一个部门周报、月报、项目总结、专题汇报，要靠人一层层整理。
现在模型可以先把原始数据、会议记录、任务状态自动归纳成结构化输出。

2. 拆解任务

过去很多中层的价值，是把一句模糊战略话术拆成几个执行动作。
现在AI已经越来越擅长把目标拆成流程、步骤、责任项和交付件。

3. 协调流程

很多流程推动型中层，本质上是“人肉工作流引擎”。
现在越来越多企业开始用系统、Agent、自动提醒、审批流和数据看板替代这件事。

4. 包装汇报

过去很多组织的中层，最核心能力不是做成事，而是把事情说成“像做成了”。
这类能力短期不会消失，但它的稀缺性会明显下降，因为模型越来越擅长做材料、做总结、做表达。

所以，AI对中层最狠的一刀，不是“老板以后不要经理了”，
而是：

老板和一线之间，原来必须通过中层才能完成的很多信息处理动作，开始有替代路径了。

这意味着，中层赖以生存的一部分“信息中介价值”，在被削弱。

三、真正危险的，不是“有中层”，而是“只有中间传递价值的中层”

说到这里，很多人会误会，好像文章在鼓吹“公司以后不要中层”。

不是这个意思。

真正的问题从来不是中层这个层级本身，
而是：

你这个中层，到底是在创造管理价值，还是只是在占据信息通道？

这两者差别非常大。

第一类中层：流程传声筒

这类人最典型的特征是：

  • • 上面说什么，就往下传
  • • 下面做什么，再往上报
  • • 主要工作是催进度、开会、拉群、收材料
  • • 离开文档、表格、汇报和会议，价值就迅速下降

这类中层，未来确实最危险。
因为AI和系统化工具正在快速吃掉这部分工作。

第二类中层：真实经营节点

这类人不一样。

他们能干的是：

  • • 判断问题轻重缓急
  • • 在复杂约束下做取舍
  • • 协调冲突资源
  • • 扛结果，而不是只扛流程
  • • 把组织目标真正变成业务结果

这种中层，短期不仅不会消失，反而会更值钱。

因为当信息处理和文档生产越来越自动化之后，
真正稀缺的，反而不是“会写会报”，而是：

  • • 会判断
  • • 会负责
  • • 会拍板
  • • 会带队
  • • 会在混乱里找到方向

所以，未来不是“中层都没了”，
而是两类中层会加速分化。

  • • 靠传递和包装活着的，会越来越难
  • • 靠判断和经营活着的，会越来越值钱

四、陈航说“没有文档”，说的其实不是文件消失，而是知识载体变了

我觉得陈航那篇里最容易被误读的一句话，就是“未来公司没有文档”。

很多人一听，第一反应是：

  • • 不可能
  • • 公司怎么可能不要文档
  • • 合同不要了？制度不要了？流程不要了？

这显然不是那个意思。

更准确地说，他讲的是：

知识和协作的核心载体，可能不再是传统文档。

过去公司为什么高度依赖文档？

因为文档是知识沉淀、信息传递、流程留痕和跨层协同的主要工具。

但AI时代，很多内容不一定再需要先写成文档再被别人读懂，
而可能变成：

  • • 结构化知识库
  • • 实时状态面板
  • • 可调用的流程系统
  • • 对话式知识接口
  • • Agent驱动的任务上下文

这意味着什么？

意味着很多原本靠文档维系的组织动作，会变成更实时、更动态、更可调用的系统能力。

文档不会彻底消失，
但它可能不再是组织运转的核心底座。

一旦这个变化成立，靠“写文档、改文档、传文档、解释文档”吃饭的那部分中间层，自然会被压缩。

五、真正要变的，不是“有没有中层”，而是组织怎么重新分工

所以，问题的关键不在于：

  • • 要不要中层
  • • 要不要扁平化
  • • 要不要砍管理层

真正关键的是：

AI进来之后，公司到底怎么重新分工。

我觉得至少会出现三个变化。

1. 信息传递职能会被压缩

原本靠层层转述完成的事情，会更多变成系统直连、数据直连、状态直连。

2. 中层必须更靠近经营结果

未来还能留下来的中层，不能只是组织缓冲垫，而必须是结果节点、判断节点、责任节点。

3. 组织会从“管人”转向“管系统 + 管例外”

很多标准化动作会交给系统和AI，
真正留给人的，会更多是非标准问题、复杂冲突、例外决策和组织韧性建设。

这意味着，未来公司不是没有管理，
而是管理会更像：

  • • 系统管理
  • • 规则管理
  • • 异常处理管理
  • • 结果管理

而不再只是传统的“层级管理”。

六、对企业真正的提醒，不是“裁中层”，而是先审一遍自己的组织结构

很多老板看完这种文章，最容易犯的错就是一句话：

那是不是可以开始砍中层了？

我劝一句，别这么快。

因为大多数公司的问题，不是中层太多，
而是：

  • • 流程本来就不清楚
  • • 组织目标本来就不透明
  • • 数据本来就不通
  • • 责任本来就不闭环

这种情况下，你把中层一砍，组织不会更高效，只会更混乱。

真正该做的，不是先砍人，
而是先问自己几个问题：

1. 哪些中层工作，本质上只是信息搬运？

这些才是最该被系统和AI替代的部分。

2. 哪些中层岗位，真正承担了判断、协调和结果责任？

这些不是该被砍，而是该被增强。

3. 公司今天有没有能力把知识、流程、状态、任务沉到系统里？

如果没有，那你嘴上讲AI，最后只会变成“多了几个工具，组织照旧低效”。

4. 你的管理动作里，哪些是在“创造结果”，哪些只是在“制造汇报”？

这个问题很多公司不敢问，但必须问。

七、最后一句判断

陈航和刘润这两篇文章，真正值得重视的，不是“中层即将消失”这句标题，
而是它们共同指向了一件事：

AI时代，公司里最先被重构的，可能不是生产岗位，而是组织中间层的信息处理、任务转译和流程协调机制。

所以，未来未必是“没有中层”，
但一定会是：

没有那么多只靠传递、包装、催办和汇报存在的中层。

真正还能留下来的中层，必须更接近：

  • • 判断
  • • 责任
  • • 经营
  • • 协同
  • • 结果

说得再狠一点：

过去很多公司的中层，是组织低效率的补丁；
未来还能活下来的中层，必须变成组织高效率的引擎。

这两者，不是一回事。
