# AI 众生相——OpenClaw 热度下的国内厂商图谱

作者：余炜勋

OpenClaw 这波热度，表面看是一个开源 Agent 项目火了。

但往深里看，它更像一张显影纸。

项目一热，厂商就下场；热度一起来，动作就分化。有人把它做成上云入口，有人接进自家模型生态，有人往企业微信、飞书、钉钉里塞，还有人顺势往行业智能体、私有化和安全可信交付上收。

**这篇文章真正想写的，不是 OpenClaw 本身，而是借它看国内厂商怎么抢 Agent 时代的入口。**

## 厂商抢的不是项目，是入口

真正让厂商敏感的，是 OpenClaw 刚好卡在几个关键位置：模型入口、云资源入口、办公消息入口、Skills 工具链入口、私有化与安全交付入口、行业解决方案入口。

谁把它接住，谁就多了一个能把模型、算力、渠道、生态和服务一起带进去的抓手。

这也是为什么，国内厂商接这波热度时，动作会长得这么不一样。

## 阿里云：上云 + 模型 + 办公入口，一体化成交

阿里云这波动作，一看就是成熟的商业化打法。

它不是只发几篇开发者文章蹭流量，而是把 OpenClaw 收进了一条完整的产品链：一端是模型——官方帮助中心已给出百炼模型接入方案，步骤清晰，不是"以后支持"，而是"现在就能接"；另一端是部署和场景——轻量应用服务器、企业微信、QQ、钉钉、飞书打包成"分钟级部署"活动页，连 Coding Plan 组合套餐也一起放上来。

阿里云不是在卖"一个开源 Agent 真有意思"，而是在卖：**云资源、模型能力、可视化配置和办公入口接入，最好一单一起成交。**

## 百度智能云：抢的是 Agent 平台和 Skills 生态入口

百度的动作更像"平台生态化"。

千帆已推出"OpenClaw 一键体验"，3 到 5 分钟部署、免费计算资源、可选百度官方 Skills、代金券配套 Coding Plan。另一侧的"快速部署 OpenClaw 定制 AI 助手"页面，把轻量服务器、Tokens 包、百度搜索、百科、学术检索、PPT 生成等官方 Skills 全部串了起来。

百度真正想做的，不是把 OpenClaw 托起来，而是把它吸进来——吸进自己的 Agent 平台、官方 Skills 和模型调用体系里。

**百度抢的，是 Agent 时代的能力分发总入口。**

## 腾讯云：先把入口卡进中国式工作流

腾讯云这波动作没那么重，但方向清晰：云应用页已上线，围绕"云上 OpenClaw 接入企业微信""接入飞书"做开发者内容。

它至少说明两件事：OpenClaw 已被包装成云上可交付的标准化应用；在中国市场，Agent 接不到企业微信、飞书这类日常工作入口，热度很难转成使用量。

腾讯云抢的位置很典型：**不是先抢模型叙事，而是先抢工作流里的触达位置。**

## 火山引擎：从部署一路铺到企业安全治理

火山引擎把 OpenClaw 做成了一条更完整的链：云服务器文档体系里单列 AI 应用类别，围绕飞书、企业微信、钉钉、QQ 给出快速部署文档；更往后的内容，已经出现"创建 OpenClaw 任务部署 AI 助手安全服务"这样的页面。

在它眼里，OpenClaw 已不是可以随便玩的个人小工具，而是会进入正式云环境、要考虑权限边界、运维流程和安全治理的问题对象。

**三段式承接：模板化部署承接开发者热度，IM 接入承接企业场景，安全和运维服务承接企业化落地需求。**

## 中科曙光：不想只吃热度流量，想吃行业落地

曙光这条线味道更重。

官网文章《5分钟部署！曙光云 × Moltbot 革新行业智能体》，讲的不是简单上云，而是和行业大模型、数据处理能力、行业智能体平台联动；不是只讲通用助手，而是直接往煤炭等垂直场景落；不是只讲部署，还讲应用镜像、私有化部署、安全可信底座。

在曙光这里，OpenClaw 被重新翻译成：**行业智能体入口、私有化载体、安全可信方案和政企交付抓手。**

它不想只吃热度流量，更想吃后面的行业落地和项目交付。

## 同一波热度，五种姿态

把五家放在一起看，局面就清楚了：

- 阿里云：上云、模型、配置、IM 接入，一体化成交
- 百度：平台入口、Skills 生态、模型和搜索能力分发
- 腾讯云：云应用和企业工作流里的触达位置
- 火山引擎：开发者部署、办公入口和企业安全治理的连续承接
- 中科曙光：行业智能体、私有化部署、安全可信和政企交付

**同一个 OpenClaw，把不同厂商最想抢的那段价值链，照得特别清楚。**

## 真正变了的，是市场对 Agent 的理解

这波厂商动作背后，对应着一个更大的变化。

过去谈 AI，谈的是模型本身：参数多大、上下文多长、推理多快。

但 OpenClaw 这类项目火起来后，市场越来越清楚：真正能形成入口、留存和商业价值的，不是孤零零的大模型，而是"模型 + 工具 + 渠道 + 任务执行 + 记忆 + 自动化"组成的完整工作流。

后面谁能接住 Agent 热度，拼的不只是模型能力，而是谁能把这些真正拧成一个闭环。

**OpenClaw 只是引子，真正的正文，是国内厂商围绕 Agent 时代入口展开的抢位。**

这只虾，已经不只是开发者在养了。厂商们，也已经下场抢位了。
