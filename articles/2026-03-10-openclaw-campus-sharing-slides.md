## Slide 2｜为什么 OpenClaw 这波这么火
**标题：**
为什么 OpenClaw 最近这么火？

**核心观点：**
OpenClaw 火，不是因为它又是一个 AI 聊天壳子，而是因为它开始像一个能长期协作的 Agent 基础设施。

**页内要点：**

- 不只是 SaaS，而是可自托管、可扩展

**一句话金句：**

## Slide 3｜OpenClaw 到底是什么
**标题：**
OpenClaw，不要把它理解成“更强聊天机器人”

**建议表达：**
把它理解成一个由以下几层组成的 Agent runtime：

**页内结构：**
- 模型层：大模型调用与路由
- 工具层：浏览器、消息、文件、shell、节点
- 记忆层：本地 memory + 长期记忆
- 自动化层：cron + heartbeat
- 扩展层：skills + plugins + scripts

**结论：**
它真正的价值，是把“回答、执行、记忆、提醒、发布”接在一起。

## Slide 4｜我们的部署原则
**标题：**
我们怎么部署：尽量官方原版，定制放外层

**核心观点：**
OpenClaw 本体尽量保持官方原版，个性化能力沉淀到 workspace、scripts、config、memory、skills。

**页内要点：**
- 官方原版 OpenClaw
- Trae + Claude Code 辅助排查和改脚本
- 不轻易魔改核心源码
- 尽量用脚本和 SOP 承载定制逻辑

**这样做的好处：**
- 升级不容易炸
- 排障边界清楚
- 更适合新手复现
- 后续更容易迁移

## Slide 5｜推荐配置方案 A
**标题：**
方案 A：本地 macOS + 国产大模型

**适合人群：**
- 想低门槛上手
- 手边有 Mac mini / MacBook
- 偏中文任务、偏本地可控

**推荐形态：**
- macOS 常开机器
- OpenClaw 官方原版
- 国产模型 / 国内兼容 OpenAI 接口
- Telegram / 飞书 / 企业微信等入口
- 本地知识库 + workspace 脚本链

**优点：**
- 上手快
- 调试体验好
- 内容生产很顺手

**注意点：**
- 休眠、权限、网络切换会影响稳定性
- 本地“装上”不代表“长期可用”

---

## Slide 6｜推荐配置方案 B
**标题：**
方案 B：海外服务器 + 国外大模型中转站

**适合人群：**
- 追求 24x7 稳定运行
- 想接更多国外模型与生态
- 有 VPS / Linux 运维基础

**推荐形态：**
- 海外 Linux 服务器
- systemd 守护 OpenClaw
- 国外模型 + 中转站统一管理
- loopback 绑定 + 环境变量隔离
- heartbeat + cron + scripts 组合运维

**优点：**
- 更适合长期在线
- 更适合多渠道、多任务自动化

**注意点：**
- 安全治理必须前置
- 不要把长期在线系统按本地玩具来配

## Slide 7｜部署完成后真正要配的东西
**标题：**
部署只是开始，真正关键的是 4 层配置

**四层结构：**
1. 模型层：别名、分流、成本控制
2. 记忆层：本地 memory / 路由脚本 / 知识库并行检索
3. 自动化层：cron / heartbeat / 巡检 / 恢复
4. 工作流层：发布链、Notion、消息投递、HTML 后处理

**结论：**
OpenClaw 能不能跑顺，不取决于“装好了没有”，而取决于这四层有没有接起来。

## Slide 8｜Skills、Plugins 和脚本分别干什么
**标题：**
不要一上来装一堆，先分清“能力扩展”和“系统扩展”

**页内结构：**
- Skills：扩展能力
  - 写作、网页抓取、Notion、记忆、浏览器自动化
- Plugins：扩展系统级集成
  - 安装 / 启用 / 禁用 / doctor / update
- Scripts：把你自己的流程固化下来
  - bootstrap / verify / publish / memory-router

**建议：**
- 新手先保留最小能力集
- 先跑通一条主链路，再慢慢加

## Slide 11｜我踩过的坑
**标题：**
真正折腾人的，不是安装，而是后面的这些坑

**页内要点：**
- 不要轻易改 OpenClaw 核心源码
- 消息链路最容易“看起来成功，实际上没送达”

**建议金句：**

## Slide 12｜怎么讲可用性、安全性和稳定性
**标题：**
如何判断一套 OpenClaw 是“能长期用”的？

**分三栏讲：**
- 可用性：任务能不能持续跑通
- 安全性：是不是按长期在线系统治理
- 稳定性：出问题能不能定位、恢复、回滚

**可落地关键词：**
- systemd
- loopback
- security audit
- plugins doctor
- cron/heartbeat 分工
- 日志 / 回执 / 备份

**收尾一句话：**
> OpenClaw 能不能长期用，不取决于你装了多少功能，而取决于你有没有把它当成一套系统来运营。
