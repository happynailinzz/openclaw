# 逮虾户的春节七天

> 一句话版：这个春节，不是我在过节，是 OpenClaw 在“上岗培训 + 连续加班 + 绩效冲刺”。

## 初一：先活下来，再优雅

第一天最朴素：先把 OpenClaw 跑起来。

- 把网关拉起来
- 把会话跑通
- 把“我是谁、你是谁、怎么说话”这些人格文件补齐（SOUL/USER/MEMORY）

当时状态就像刚开工的厨房：锅有了，火也有了，但菜谱还在脑子里。

## 初二：从“能用”到“不翻车”

第二天开始补运维地基，不然早晚翻车：

- 安全审计跑起来
- 定时任务（cron）开起来
- 心跳巡检节奏固定下来
- 配置和日志开始有归档

那一刻的感受是：终于不是“临时搭子”，开始像“可值夜班”的系统了。

## 初三：技能大采购，拒绝裸奔

把核心技能链路装齐，尤其是内容生产和检索相关：

- 搜索、网页抓取、知识库、文档处理
- Notion 集成
- 去 AI 味写作优化

本质上做了一件事：把“会聊天”升级成“会干活”。

## 初四：给它灵魂，不当客服腔

这一趴很关键：把输出风格从“企业客服公文体”改成“靠谱同事体”。

- 结论先行
- 少废话
- 该拦坑就拦坑
- 保留一点人味儿和幽默感

效果非常直观：同样一件事，它不再像在写周报，而是像在跟你并肩做项目。

## 初五：写作流水线打通（知识库优先）

公众号工作流按你定的规则落地：

1. 先查 `/opt/openclaw/knowledge/secendME`
2. 再用 MEMORY/会话记忆
3. 最后才外部资料

并且输出里要标注来源类别（知识库/延伸/外部）。

这一步的价值是：内容开始“像你写的”，而不是“像互联网平均值”。

## 初六：Notion 入库和分发闭环

把“写完就散”改成“写完入库”：

- 公众号选题与草稿默认进 Notion「文章选题草稿箱」
- 文章可导出 HTML
- 可发 Telegram 做交付/预览

到这一步，流程从“写作动作”变成了“生产系统”。

## 初七：生产化体检，交卷

最后一天做的是硬核但不性感的事：生产化加固。

### 已经完成/落地的关键能力

- OpenClaw 升级到 **2026.2.24**
- 网关稳定运行（loopback、本机可达）
- 定时任务 **6 条**持续运行，最近状态均为 **ok**
- 形成两个可复用脚本：
  - `scripts/bootstrap-openclaw.sh`（一键恢复部署骨架）
  - `scripts/verify-openclaw.sh`（一键体检）
- 内容侧形成“Markdown -> HTML -> 推送/入库”的闭环

### 还没彻底补齐的点

- embeddings 仍未启用（当前 memory 检索还是 `fts-only`）
- Notion API key 当前配置异常，需修正后恢复 API 直写稳定性

这两个点，一个影响“检索聪明度”，一个影响“入库稳定性”，都是下一阶段优先级最高。

---

## 这七天到底实现了什么？

如果用人话总结：

我们不是“部署了一个 AI 工具”，而是把它调成了一个**可持续交付的数字同事**。

它现在具备：

- 会查：检索/抓取/知识库调用
- 会写：选题、策划、改稿、去 AI 味
- 会跑：定时任务、巡检、状态回报
- 会交付：HTML、消息推送、Notion 入库（修完 key 后可持续稳定）

## Token 和成果，算账版

按当前会话统计口径看（去重后）：

- 累计 token 量级约 **26 万+**（含多条任务会话）
- 若把 run 镜像会话也并入，账面可见总量约 **47 万+**

产出层面：

- 已沉淀多篇公众号文章与 HTML 成稿
- 运维脚本 2 个（部署 + 体检）
- 稳定 cron 任务 6 条
- 多次健康检查/版本检查常态化

一句话：Token 没白烧，换来的是“从一次性演示到可复用系统”。

---

## 结尾：逮虾户，不是飙车，是稳交付

《逮虾户的春节七天》真正的后劲，不是“我把 OpenClaw 装上了”，而是：

**我把它从“会说”调成了“会做、会交付、可复盘”。**

下一步就两件事：

1. 把 embeddings 补上，让记忆检索从“关键词命中”升级到“语义召回”
2. 把 Notion key 配置修好，让入库链路彻底稳定

做完这两步，这套系统基本就能长期跑业务了。

—— 春节收官，交卷。
