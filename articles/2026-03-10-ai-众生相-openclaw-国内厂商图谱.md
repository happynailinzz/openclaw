# AI 众生相——OpenClaw 热度下的国内厂商图谱

作者：余炜勋

最近 OpenClaw 这波热度，表面上看，是一个开源 Agent 项目突然火了。

但真要往深里看，它更像一张显影纸。

项目一热，厂商就下场；热度一起来，动作就分化。有人忙着把它做成上云入口，有人忙着把它接进自家模型生态，有人忙着往企业微信、飞书、钉钉、QQ 这些工作入口里塞，还有人干脆顺势把它往行业智能体、私有化和安全可信交付上收。

所以，这篇文章真正想写的，不是 OpenClaw 本身。

**这是一篇借 OpenClaw 看国内厂商怎么抢 Agent 时代入口的文章。**

说白了，同样是一波 OpenClaw 热度，不同厂商盯上的根本不是同一个东西。

有人盯的是云主机，有人盯的是模型调用，有人盯的是技能生态，有人盯的是企业办公入口，还有人盯的是政企项目、行业交付和私有化部署。

OpenClaw 只是那个把局面照亮的触发点。

## 一，厂商抢的不是 OpenClaw，而是 OpenClaw 背后的入口

如果只把 OpenClaw 当成一个会调工具、会跑任务、能接消息渠道的开源 Agent，你看到的还只是表层。

真正让厂商敏感的，是它刚好卡在几个关键位置上：

- 模型入口
- 云资源入口
- 办公消息入口
- Skills / 工具链入口
- 私有化与安全交付入口
- 行业解决方案入口

也就是说，谁把 OpenClaw 接住，谁就不只是多了一个热门项目，而是多了一个能把模型、算力、渠道、生态和服务一起带进去的抓手。

这也是为什么，国内厂商接这波热度的时候，动作会长得这么不一样。

它们不是在重复做一件事，而是在把同一个热点，往自己最擅长、也最想变现的那段价值链里收。

## 二，阿里云：要的不是话题热度，而是“上云 + 模型 + 办公入口”一体化成交

阿里云这波动作，属于一看就很成熟的商业化打法。

它不是只发几篇开发者文章蹭流量，而是很快把 OpenClaw 收进了一条完整的产品链。

一端是模型。

阿里云已经在官方帮助中心里明确给出 OpenClaw 对接百炼模型的接入方案，模型列表、Base URL、API Key、安装和配置步骤都写得很清楚。这不是“以后可以支持”，而是“现在就能接”。

另一端是部署和场景。

阿里云又很快把它做成了“不用写代码，分钟级部署 OpenClaw”的活动页。页面里讲得非常直接：本地部署容易断网、休眠、更新麻烦，真正适合长期在线运行的，是轻量应用服务器。再往下推，它又把企业微信、QQ、钉钉、飞书这些国内主流工作入口打包进去，连 Coding Plan 这样的组合套餐也一起放上来。

这套动作拼起来，你就能看明白阿里云在干什么了。

它不是在卖“一个开源 Agent 真有意思”，而是在卖：

**云资源、模型能力、可视化配置和办公入口接入，最好一单一起成交。**

所以阿里云接这波热度，最核心的目标不是讨论 Agent，而是把 OpenClaw 变成一条新的开发者上云路径。

## 三，百度智能云：借 OpenClaw 抢的，是 Agent 平台和 Skills 生态入口

如果说阿里云的路数偏“云产品化”，那百度的动作就更像“平台生态化”。

百度千帆已经推出了“千帆 OpenClaw 一键体验”，强调 3 到 5 分钟完成部署、免费计算资源支持、可选百度官方 Skills，还会给代金券，配合 Coding Plan 一起降低试用成本。

这还不算完。

百度智能云另一侧的“快速部署 OpenClaw 定制 AI 助手”页面，已经不只是告诉你怎么装，而是把轻量服务器、BCC、Tokens 包、百度搜索、百科、学术检索、PPT 生成等官方 Skills 全部串了起来。

这说明百度真正想做的，不是把 OpenClaw 托起来，而是把它吸进来。

吸进哪？

吸进自己的 Agent 平台、官方 Skills 和模型调用体系里。

换句话说，在百度这儿，OpenClaw 更像一个前台入口。你是从它进来的，但你最后接触到的，会是千帆、百度搜索、百科、学术、PPT、深度研究 Agent 这一整套百度自己的能力体系。

所以百度抢的，不是一个单点项目，而是：

**谁来做 Agent 时代的能力分发总入口。**

这也是为什么，百度这条线的味道，明显比“部署教程”更重、更平台化。

## 四，腾讯云：先把入口卡进中国式工作流里

腾讯云这波动作没有阿里和百度那么重，但方向并不模糊。

它已经有了 OpenClaw 的云应用页，也已经开始围绕“云上 OpenClaw 接入企业微信”“接入飞书”这类场景做开发者内容。

动作不算花，但意思已经到了。

它至少说明两件事：

第一，腾讯云已经承认 OpenClaw 不是一个只能在个人电脑上玩的项目，它可以被包装成云上可交付的标准化应用；

第二，在中国市场里，Agent 如果接不到企业微信、飞书这类日常工作入口里，热度很难真正转成使用量。

所以腾讯云当前抢的位置，其实很典型，也很腾讯：

**不是急着先抢模型叙事，而是先抢工作流里的触达位置。**

这条路听起来没那么炸裂，但很务实。

因为真正能形成留存的，不一定是参数最强的模型，而往往是谁先嵌进了用户每天都会打开的工作入口。

## 五，火山引擎：从开发者部署一路铺到企业安全治理

火山引擎的动作也很有意思，因为它把 OpenClaw 这件事做成了一条更完整的链。

它在云服务器文档体系里，已经把 OpenClaw（原 Moltbot）单列成一个 AI 应用类别，围绕飞书、企业微信、钉钉、QQ 这些典型办公入口给出快速部署文档。

这一步说明的是：它已经把 OpenClaw 看成一个值得标准化承接的应用场景。

但更值得看的是后半段。

火山引擎并没有止步于“教你装”，而是继续往安全和运维方向延伸，甚至出现了“创建 OpenClaw（Linux）任务部署 AI 助手安全服务”这样的内容。

这背后的意思非常明确：

在它眼里，OpenClaw 已经不是一个可以随便玩的个人小工具，而是一个会进入正式云环境、要考虑权限边界、运维流程和安全治理的问题对象。

所以火山引擎这一套，本质上是在做三段式承接：

- 用模板化部署承接开发者热度
- 用飞书/企微/钉钉/QQ 承接企业使用场景
- 再用安全和运维服务承接企业化落地需求

这条路的商业意味，其实已经很浓了。

## 六，中科曙光：最不想只做“热度流量”的那家

如果说前面几家厂商，更多还是围绕云、模型、入口和生态做动作，那中科曙光这条线的味道就更重了。

因为它已经明显不满足于“我也支持 OpenClaw”这种姿态，而是在把它往行业智能体和政企交付上重新定义。

曙光官网有一篇文章，标题就很直白：《5分钟部署！曙光云 × Moltbot 革新行业智能体》。

但真正值得看的不是这个标题，而是它文章里那套表达方式：不是简单把 Moltbot 放上云，而是和曙光云现有的行业大模型、数据处理能力、行业智能体平台联动；不是只讲通用助手，而是直接往煤炭等行业垂直场景里落；不是只讲部署，还讲应用镜像、私有化部署、安全可信底座。

这就说明，在曙光这里，OpenClaw 已经不是“热门开源项目”，而是被重新翻译成了另一种语言：

**行业智能体入口、私有化载体、安全可信方案和政企交付抓手。**

这和前面几家完全不是一个收口方式。

阿里、百度、腾讯、火山更多是在接开发者、接平台、接入口；而曙光更像是在说：这波热度最后会走向行业客户、走向政企项目、走向私有化和安全可信，而这些地方，正好是它的主场。

所以如果把几家厂商放在一起看，曙光这条线最不一样的地方就在于：

**它不想只吃热度流量，它更想吃后面的行业落地和项目交付。**

## 七，同一波热度，为什么会长出五种完全不同的姿态

把这五家放在一起看，局面就清楚了。

它们并不是在做同一道题，而是在借同一个热点，把自己最核心的业务重新前置。

阿里云要的是：上云、模型、配置、IM 接入，一体化成交；
百度要的是：平台入口、Skills 生态、模型和搜索能力分发；
腾讯云要的是：云应用和企业工作流里的触达位置；
火山引擎要的是：开发者部署、办公入口和企业安全治理的连续承接；
中科曙光要的是：行业智能体、私有化部署、安全可信和政企交付。

看到这儿，其实就明白了。

所谓“AI 众生相”，并不是说大家都看懂了 OpenClaw，而是说：

**同一个 OpenClaw，把不同厂商最想抢的那段价值链，照得特别清楚。**

有人先抢开发者；
有人先抢模型调用；
有人先抢企业办公入口；
有人先抢平台生态；
有人直接去抢行业项目和私有化交付。

这波热度一来，厂商各自的站位就全暴露出来了。

## 八，真正变了的，不是一个工具，而是市场对 Agent 的理解

如果再往前看一步，这波厂商动作背后，其实对应着一个更大的变化。

过去大家谈 AI，谈得最多的是模型本身：参数多大、上下文多长、推理多快、能力多强。

但 OpenClaw 这类项目火起来之后，市场正在越来越清楚地意识到：

真正能形成入口、形成留存、形成商业价值的，不是一个孤零零的大模型，而是“模型 + 工具 + 渠道 + 任务执行 + 记忆 + 自动化”组成的完整工作流。

这意味着，后面谁能真正接住 Agent 热度，拼的就不只是模型能力了。

拼的是谁能把模型、云资源、消息入口、工具链、安全治理和行业场景真正拧成一个闭环。

从这个角度看，OpenClaw 火不火只是表面现象。

真正值得重视的是，它正在逼着国内厂商回答一个更现实的问题：

**Agent 时代来了以后，你到底准备卖什么？**

卖部署，卖模型，卖生态，卖入口，卖安全，还是卖行业方案？

每家厂商都已经给出了自己的答案。

## 九，最后一句：OpenClaw 只是引子，抢位才是正文

如果把这波现象只理解成“一个开源项目火了”，那其实看浅了。

更准确的说法应该是：

**OpenClaw 只是一个引子，真正的正文，是国内厂商围绕 Agent 时代入口展开的抢位。**

今天最先把动作做出来的，是阿里云、百度智能云、腾讯云、火山引擎和中科曙光。

后面围绕算力、一体机、私有化、安全、园区和地方配套，大概率还会冒出更多角色、更多打法、更多更重的商业动作。

但至少到现在，第一批已经落地的厂商图谱已经出来了。

OpenClaw 这只虾，已经不只是开发者在养了。

厂商们，也已经下场抢位了。
