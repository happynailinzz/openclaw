更新时间：2026-03-03 11:28 UTC
用途：将能力以树结构管理，并按价值函数驱动进化与修剪。

## A. 低层能力（基础操作 / 稳定工具使用）

### A1. 文档抓取与读取（Web/本地）
- 能力名称：链接与文件内容提取
- 输入条件：URL 或本地路径可访问
- 输出结果：可读文本（markdown/text）
- 成功前提：目标可达、无强登录/验证码拦截
- 失败边界：风控/验证码导致正文不可得

### A2. 规则文件读写与版本化
- 能力名称：规则落地与可追踪修改
- 输入条件：规则文件路径存在、git 可用
- 输出结果：规则变更 + 提交记录
- 成功前提：写权限正常，提交信息清晰
- 失败边界：写入失败、提交冲突或无权限

### A3. 本地知识库检索（Obsidian/Markdown）
- 能力名称：知识库定位与命中验证
- 输入条件：库路径已知、文本可检索
- 输出结果：命中列表或未命中结论
- 成功前提：文件系统可读、关键词合理
- 失败边界：关键词缺失/库外内容导致误判

### A4. 系统资源诊断与清理
- 能力名称：内存/磁盘占用定位与释放
- 输入条件：系统可执行 `ps`/`ss`/`du`/`free`
- 输出结果：占用大户清单 + 可释放项确认 + 执行后内存/磁盘变化数据
- 成功前提：进程可 kill，文件可删，无重要服务依赖
- 失败边界：进程是关键服务依赖、文件被占用、权限不足
- 实战验证（2026-03-03）：释放内存 ~1.6G（Chrome 僵尸进程）+ 磁盘 ~1G（qmd 模型 + Playwright）

### A5. Docker 容器网络绑定收口
- 能力名称：将对外暴露端口改绑到 127.0.0.1
- 输入条件：容器名已知、`docker inspect` 可获取启动参数
- 输出结果：容器重建，仅本地可访问，原有 volume/env/restart 策略保留
- 成功前提：容器可停止重建，服务本地调用路径不变
- 失败边界：容器依赖外网访问、重建后配置丢失
- 实战验证（2026-03-03）：图片代理 8317 从 `0.0.0.0` 收口到 `127.0.0.1`，本地调用正常

## B. 中层能力（可复用流程 / 策略模式）

### B1. 规则合并流程（新指令→兼容规则）
- 能力名称：增量规则合并
- 输入条件：已有基线规则 + 用户新增规则
- 输出结果：无冲突的兼容版规则
- 成功前提：明确优先级与冲突处理顺序
- 失败边界：语义冲突未识别，导致执行歧义

### B2. 能力准入评估流程（价值函数门禁）
- 能力名称：候选能力入树评估
- 输入条件：候选能力描述 + 5维评估
- 输出结果：入树 / 观察 / 拒绝
- 成功前提：有可验证指标（复用、失败率、成本）
- 失败边界：仅主观判断，缺少验证数据

### B3. 失败回退流程（反劣化与反进化锁）
- 能力名称：策略回滚与风险止损
- 输入条件：出现成功率下降或确定性下降
- 输出结果：回滚到上个稳定版本
- 成功前提：有稳定版本可回退
- 失败边界：未保留回滚点导致不可逆

### B4. 错误自愈与自省调试链路（新增）
- 能力名称：失败自动诊断与修复建议
- 输入条件：工具调用错误、执行中断、结果缺失
- 输出结果：错误分类（暂时/永久/权限/配置）+ 修复动作 + 是否回退
- 成功前提：有结构化错误上下文可读
- 失败边界：错误不可复现或上下文缺失

### B5. 外部请求稳定性治理
- 能力名称：HTTP 重试/退避/超时治理
- 输入条件：涉及外部 HTTP 调用
- 输出结果：成功响应或确定性失败（含重试轨迹）
- 成功前提：可配置 retry/backoff/timeout 上限
- 失败边界：上游长期不可用或永久业务错误
- 实战验证（2026-03-03）：docker 容器重建后 curl 返回 `Missing API key` = 服务存活信号

### B6. 多工具 Chrome 路径统一流程
- 能力名称：通过环境变量统一多个 skill 的浏览器路径
- 输入条件：多个 skill 各自用不同 Chrome 二进制，均支持环境变量覆盖
- 输出结果：`.env` 写入所有 skill 的 `*_CHROME_PATH` 变量，指向同一二进制；冗余 Chrome 缓存删除
- 成功前提：各 skill 有对应环境变量覆盖入口；目标 Chrome 版本兼容
- 失败边界：某 skill 无环境变量覆盖、Chrome 版本差异导致兼容问题
- 实战验证（2026-03-03）：5个 skill 统一到 Puppeteer Chrome，删除 Playwright 节省 622MB

### B7. 记忆三源检索流程
- 能力名称：本地日志 + qmd 知识库 + Memos 并发检索
- 输入条件：query 文本；qmd 已建索引；Memos API 可用
- 输出结果：结构化 JSON（local_hits + qmd_hits + memos_hits）
- 成功前提：三源至少一源可达；`memory-retrieval-router.py` 可执行
- 失败边界：qmd 未建索引（无命中）；Memos 不可达（降级为本地+qmd双源）
- 触发命令：`cd /root/.openclaw/workspace && python3 scripts/memory-retrieval-router.py "<query>"`
- 实战验证（2026-03-03）：三源并发检索"源网荷储 数字化"，本地2条 + qmd 1条 + Memos 1条全部命中

### B8. 系统安全加固标准流程
- 能力名称：单机 Linux 最小化攻击面治理
- 输入条件：`ss`/`ps`/`free` 可用；有 root 权限
- 输出结果：Swap 启用、高风险端口收口、敏感服务审查报告
- 成功前提：服务可重启/重建；fstab 可写
- 失败边界：容器编排复杂；端口收口影响正在使用的外部服务
- 标准动作顺序：① 查内存/Swap → ② 扫外网端口 → ③ 加 Swap → ④ 收口高风险端口
- 实战验证（2026-03-03）：Swap 2G 上线 + 8317 收口，整体风险评分从 5/10 → 7/10

## C. 高层能力（问题分解 / 决策范式）

### C1. 结构化进化决策
- 能力名称：树内路径优先决策
- 输入条件：新任务目标 + 现有能力树
- 输出结果：优先复用路径 + 必要新分支说明
- 成功前提：树结构清晰、节点定义完整
- 失败边界：树信息失真，导致重复造轮子

### C2. 价值驱动演化决策
- 能力名称：以长期效用选择进化方向
- 输入条件：候选能力集合 + 价值函数评分
- 输出结果：高价值能力进化队列
- 成功前提：可量化对比（成本、成功率、复用）
- 失败边界：为“看起来高级”引入复杂度
