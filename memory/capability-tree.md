# Capability Tree（v0.1）

更新时间：2026-03-01 04:24 UTC
用途：将能力以树结构管理，并按价值函数驱动进化与修剪。

---

## A. 低层能力（基础操作 / 稳定工具使用）

### A1. 文档抓取与读取（Web/本地）
- 能力名称：链接与文件内容提取
- 输入条件：URL 或本地路径可访问
- 输出结果：可读文本（markdown/text）
- 成功前提：目标可达、无强登录/验证码拦截
- 失败边界：风控/验证码导致正文不可得

### A2. 规则文件读写与版本化
- 能力名称：规则落地与可追踪修改
- 输入条件：规则文件路径存在、git 可用
- 输出结果：规则变更 + 提交记录
- 成功前提：写权限正常，提交信息清晰
- 失败边界：写入失败、提交冲突或无权限

### A3. 本地知识库检索（Obsidian/Markdown）
- 能力名称：知识库定位与命中验证
- 输入条件：库路径已知、文本可检索
- 输出结果：命中列表或未命中结论
- 成功前提：文件系统可读、关键词合理
- 失败边界：关键词缺失/库外内容导致误判

## B. 中层能力（可复用流程 / 策略模式）

### B1. 规则合并流程（新指令→兼容规则）
- 能力名称：增量规则合并
- 输入条件：已有基线规则 + 用户新增规则
- 输出结果：无冲突的兼容版规则
- 成功前提：明确优先级与冲突处理顺序
- 失败边界：语义冲突未识别，导致执行歧义

### B2. 能力准入评估流程（价值函数门禁）
- 能力名称：候选能力入树评估
- 输入条件：候选能力描述 + 5维评估
- 输出结果：入树 / 观察 / 拒绝
- 成功前提：有可验证指标（复用、失败率、成本）
- 失败边界：仅主观判断，缺少验证数据

### B3. 失败回退流程（反劣化与反进化锁）
- 能力名称：策略回滚与风险止损
- 输入条件：出现成功率下降或确定性下降
- 输出结果：回滚到上个稳定版本
- 成功前提：有稳定版本可回退
- 失败边界：未保留回滚点导致不可逆

### B4. 错误自愈与自省调试链路（新增）
- 能力名称：失败自动诊断与修复建议
- 输入条件：工具调用错误、执行中断、结果缺失
- 输出结果：错误分类（暂时/永久/权限/配置）+ 修复动作 + 是否回退
- 成功前提：有结构化错误上下文可读
- 失败边界：错误不可复现或上下文缺失

### B5. 外部请求稳定性治理（新增）
- 能力名称：HTTP 重试/退避/超时治理
- 输入条件：涉及外部 HTTP 调用
- 输出结果：成功响应或确定性失败（含重试轨迹）
- 成功前提：可配置 retry/backoff/timeout 上限
- 失败边界：上游长期不可用或永久业务错误

## C. 高层能力（问题分解 / 决策范式）

### C1. 结构化进化决策
- 能力名称：树内路径优先决策
- 输入条件：新任务目标 + 现有能力树
- 输出结果：优先复用路径 + 必要新分支说明
- 成功前提：树结构清晰、节点定义完整
- 失败边界：树信息失真，导致重复造轮子

### C2. 价值驱动演化决策
- 能力名称：以长期效用选择进化方向
- 输入条件：候选能力集合 + 价值函数评分
- 输出结果：高价值能力进化队列
- 成功前提：可量化对比（成本、成功率、复用）
- 失败边界：为“看起来高级”引入复杂度
