# Token Budget Policy（OpenClaw）

更新时间：2026-03-01

## 预算上限
- 检索上下文：<= 1800 tokens
- 系统/规则上下文：<= 1200 tokens
- 输出预留：>= 800 tokens

## 检索裁剪规则
1. 热记忆优先：`memory/YYYY-MM-DD.md`（今天/昨天）+ `MEMORY.md`
2. 不足再查 Memos（`search/memory`）
3. 每次最多带入 3-5 条证据
4. 单条证据默认 <= 500 chars

## 长文处理
- 原文超过 2k 行：先摘要（200-300字）再决定是否展开
- 外部网页：优先关键段落，不整页注入

## 回退
- 若结果质量下降：临时放宽到 6-8 条证据
- 若 token 爆炸：降到 2-3 条证据并缩短单条长度
